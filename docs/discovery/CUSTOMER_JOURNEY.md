# Customer Journey

## Journey A — первый пользовательский опыт

| Этап | Действие пользователя | Ожидание | Риск | Требование продукта |
|---|---|---|---|---|
| Обнаружение | Видит продукт или рекомендацию | Быстро понять выгоду | Обещание выглядит как очередная скидка | Простое сообщение: «Покупай как обычно. Получай больше» |
| Первый вход | Открывает приложение | Начать без обучения | Длинная регистрация | Сканирование до полной настройки профиля |
| Скан | Фотографирует чек | Чек примут с первого раза | Плохое изображение | Контроль качества и подсказки |
| Ожидание | Чек обрабатывается | Быстрый результат | Неясный или долгий процесс | Статус и предварительный результат |
| Результат | Получает баллы и билет | Почувствовать дополнительную ценность | Награда слишком мала или непонятна | Конкретное начисление и изменение целей |
| Цель | Выбирает награду | Понять смысл следующего скана | Каталог неинтересен | Награды с низким порогом и реальной доступностью |
| Возврат | Делает новую покупку | Быстро приблизиться к цели | Забыл о продукте | Уведомление о прогрессе, а не спам |

## Journey B — накопленные чеки за день

1. Пользователь сохраняет несколько чеков.
2. Открывает приложение в удобный момент.
3. Запускает пакетное сканирование.
4. Добавляет чеки последовательно.
5. Видит очередь обработки.
6. Получает отдельный результат каждого чека.
7. Получает итог сессии:
   - общие баллы;
   - новые билеты;
   - прогресс к награде;
   - выполненную возможность дня.

### Критический момент

Пакетный сценарий не должен заставлять ждать завершения обработки каждого чека перед следующим снимком.

## Journey C — партнерское предложение

1. Пользователь видит возможность дня.
2. Понимает условие, срок и выгоду.
3. Принимает решение посетить партнера.
4. Совершает покупку.
5. Сканирует чек.
6. Система связывает чек с кампанией.
7. Пользователь получает усиленную ценность.
8. Бизнес видит подтвержденную покупку.

### Требование доверия

Пользователь заранее знает, при каких условиях предложение будет засчитано.

## Journey D — обмен награды

1. Пользователь достигает цели.
2. Получает уведомление.
3. Открывает карточку награды.
4. Проверяет условия и срок.
5. Подтверждает обмен.
6. Баллы резервируются.
7. Получает код или QR.
8. Бизнес подтверждает использование.
9. Пользователь видит завершенную операцию.

### Failure recovery

При отказе бизнеса пользователь может быстро открыть спор, не теряя историю операции.

## Journey E — бизнес-партнер

1. Бизнес узнает о платформе.
2. Видит модель оплаты за измеримый результат.
3. Регистрируется и подтверждает организацию.
4. Создает первую простую награду.
5. Настраивает повышенные баллы или предложение дня.
6. Кампания получает охват.
7. Появляются подтвержденные покупки.
8. Бизнес видит новых и повторных клиентов.
9. Сравнивает стоимость результата.
10. Запускает следующую кампанию.

## Moments of Truth

- первый подтвержденный чек;
- первая видимая цель;
- первый обмен награды;
- первый розыгрыш;
- первая подтвержденная покупка для бизнеса;
- первый повторный запуск кампании партнером.
