# Value Proposition

## Для пользователя

### Functional value

- превратить любой подтвержденный чек в баллы и билет;
- объединить выгоду от разных покупок в одном месте;
- видеть прогресс к выбранной награде;
- получать актуальную возможность сегодня;
- оставлять подтвержденные отзывы.

### Emotional value

> Я получил больше, чем заплатил.

Пользователь не чувствует, что после оплаты ценность закончилась.

### Social value — поздний этап

- подтвержденная репутация;
- семейные и командные цели;
- участие в локальных сообществах;
- поддержка локального бизнеса.

## Для бизнеса

### Functional value

- продвижение внутри активной аудитории;
- подтвержденные покупки вместо одних показов;
- новые и повторные клиенты;
- программа лояльности без собственной разработки;
- отзывы реальных покупателей;
- измерение ROI кампаний.

### Economic value

Бизнес получает возможность платить за результат ближе к реальной выручке:

```text
Охват -> Визит -> Подтвержденная покупка -> Повторный визит
```

### Operational value

- быстрый запуск награды;
- отсутствие обязательной POS-интеграции в MVP;
- управление филиалами;
- простой кабинет;
- понятные отчеты.

## Для платформы

- сеть подтвержденных покупок;
- рост аудитории через любые чеки;
- рост ценности через партнеров;
- накопление доверенной истории взаимодействий;
- рекламная атрибуция на уровне офлайн-покупки.

## Positioning Statement

Для людей, которые хотят получать больше от привычных покупок, Receipt Network превращает любой подтвержденный чек в баллы, шанс и прогресс к реальной награде. В отличие от разрозненных программ лояльности, продукт работает между разными бизнесами и не требует менять повседневные привычки.

Для локальных бизнесов Receipt Network — это измеримый канал привлечения и возврата клиентов, который связывает продвижение с подтвержденной офлайн-покупкой.

## Message hierarchy

1. Покупай как обычно. Получай больше с каждой покупки.
2. Сканируй любой чек.
3. Получай баллы и билеты.
4. Приближайся к выбранной награде.
5. У партнеров получай больше.
