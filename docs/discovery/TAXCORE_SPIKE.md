# TaxCore / PURS Verification Spike

**Дата:** 2026-07-16
**Статус:** проверено вживую на реальном чеке
**Связано с:** ADR-P-013 (QR-first), IMPLEMENTATION_PLAN §4 (критический путь), OPEN_QUESTIONS (QR/фискальные данные)
**Скрипт:** `apps/api/scripts/taxcore-spike.ts`

## Что проверяли

Главное допущение продукта: можем ли мы из QR на сербском фискальном чеке получить данные, подтверждённые государством, без OCR и скрейпинга.

## Механика (подтверждена)

1. QR-код чека кодирует URL вида `https://suf.purs.gov.rs/v/?vl=<payload>`.
2. GET-запрос на **тот же URL** с заголовком `Accept: application/json` возвращает чек в JSON (контент-негоциация — не отдельный API-путь).
3. HTTP 200 → тело JSON со структурой:

```
{
  isValid: boolean,          // подтверждение подлинности государством
  invoiceRequest: {...},
  invoiceResult: {           // структурированные поля
    invoiceNumber, sdcTime,  // номер чека, дата/время
    totalAmount, ...
  },
  journal: "…"               // полный чек текстом: продавец, PIB, адрес, позиции
}
```

## Результат живого теста

Реальный чек (пример из библиотеки-парсера), запрос вернул HTTP 200:

- `isValid`: **true** (подтверждено государством)
- `invoiceNumber`: `VBMHX9SX-W6UBPZO0-76722`
- `sdcTime` (дата): `2023-01-03T10:15:33Z`
- `totalAmount`: `1500.0` RSD
- продавец (из journal): `RESTAURANT PLEASURE PARK ČAIR`, PIB `112392483`, адрес `ИX БРИГАДЕ 27 Л1, Ниш-Медијана`
- позиции (из journal): Burito sa piletinom 650, Grcki omlet 490, Elixir mix 0.33l 360

## Выводы

- **QR-first реализуем.** Подлинность, номер, дата и сумма приходят структурированно и подтверждены государством — это снимает большую часть fraud-риска «поддельный чек».
- **Продавец и позиции — в текстовом `journal`.** Нужен парсер журнала (грамматика уже есть в open-source PHP-библиотеке turanjanin/serbian-fiscal-receipts-parser — портировать в Recognition-домен).
- **OCR остаётся только fallback** — для нечитаемого QR и нефискальных чеков (ADR-P-013 подтверждён практикой).

## Что ещё открыто (для Phase 0)

- Лимиты и условия доступа к endpoint при массовом использовании (rate limits, нужна ли регистрация ESD/ESIR).
- Всегда ли `journal` присутствует (разные версии ЕСИР/типы чеков).
- Офлайн-декод `vl` (buyerId, receiptNumber, counter, date) — для быстрой предпроверки и антидубликата до сетевого запроса.
- Покрытие нефискальных чеков и услуг вне схемы TaxCore.

## Как запустить

```bash
npx tsx apps/api/scripts/taxcore-spike.ts "https://suf.purs.gov.rs/v/?vl=..."
```

Скрипт валидирует домен, дёргает JSON-API, печатает подтверждение + продавца + позиции. Зависимостей нет (Node 20+ global fetch).
