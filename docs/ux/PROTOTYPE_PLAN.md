# Prototype Plan

## Objective

Create a clickable low-fidelity prototype that tests comprehension and motivation before visual design or engineering.

## Prototype Scope

### Flow A — First Receipt

1. Onboarding.
2. Camera permission.
3. Receipt capture.
4. Processing.
5. Registration.
6. Receipt result.
7. Home with three value horizons.

### Flow B — Select Reward Goal

1. Home without active goal.
2. Rewards catalogue.
3. Reward details.
4. Set goal.
5. Home with progress.

### Flow C — Batch Scan

1. Scanner.
2. Capture first receipt.
3. Add another.
4. Queue with one low-quality receipt.
5. Rescan.
6. Session results.

### Flow D — Daily Opportunity

1. Home.
2. Opportunity details.
3. Partner page.
4. Simulated purchase and receipt scan.
5. Enhanced result.

### Flow E — Redeem Reward

1. Goal completed.
2. Reward details.
3. Confirmation.
4. QR/code.
5. Redeemed state.

## Research Questions

- Does the promise make sense without explanation?
- Do users understand that any receipt has value?
- Do they distinguish points from lottery tickets?
- Can they explain the three Home cards?
- Do they know what to do after a rejected receipt?
- Does a selected reward create desire to scan again?
- Does the daily opportunity feel useful rather than advertising?

## Test Tasks

- Scan your first receipt.
- Explain what you received.
- Choose a reward you want.
- Find how many tickets you have.
- Find an offer available today.
- Add two more receipts in one session.
- Use a completed reward.

## Success Criteria

- 80% complete first scan without help.
- 80% correctly describe points and tickets as separate.
- 70% notice all three value horizons on Home.
- 70% can set a reward goal.
- 80% recover from a simulated scan-quality error.
- majority describe the product as receiving extra value, not merely storing receipts.

## Prototype Fidelity

Use grayscale layouts with real Serbian or bilingual copy, realistic receipt amounts and plausible rewards. Do not test branding before testing comprehension.

## Outputs

- task completion results;
- confusion log;
- copy changes;
- navigation changes;
- updated assumptions;
- recommendation to proceed, revise or simplify.
