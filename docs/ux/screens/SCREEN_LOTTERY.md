# Screen Specification — Lottery

## Цель

Показать пользователю прозрачный прогресс участия и понятные правила розыгрыша.

Реализуется как вкладка «Срёчки» внутри экрана «Награды» (см. `SCREEN_REWARDS.md`, FR-6/FR-7).

## Основные блоки

- активный розыгрыш текущего периода (в MVP — ежемесячный);
- обратный отсчёт до розыгрыша;
- призы;
- количество билетов пользователя в текущем периоде;
- список билетов (лотерейных номеров);
- правила;
- прошлые результаты;
- история выигрышей пользователя.

## Ticket Card

- Ticket ID;
- связанный чек;
- дата выдачи;
- статус;
- розыгрыш.

## Состояния

- no active draw;
- active draw;
- draw locked;
- drawing in progress;
- winner;
- not selected;
- prize claim required;
- prize claimed;
- ticket invalidated;
- results delayed.

## Прозрачность

Пользователь должен видеть:

- почему билет создан;
- в каком розыгрыше он участвует;
- когда список участников блокируется;
- срок получения приза;
- причину аннулирования, если оно произошло.

## Аналитика

- `lottery_viewed`;
- `ticket_details_viewed`;
- `lottery_rules_viewed`;
- `draw_result_viewed`;
- `prize_claim_started`;
- `prize_claim_completed`.

## UX-правила

- не обещать выигрыш;
- не скрывать условия и сроки;
- явно показывать, что билеты действуют только в текущем периоде и не переносятся на следующий (ADR-P-014);
- баллы и билеты визуально не смешиваются;
- результаты должны оставаться доступными в истории.
