# Screen Specification — Home

## Цель

Показать текущую ценность и дать самый быстрый путь к следующему скану.

## Главный принцип

На экране одновременно не более трех мотивационных целей.

## Иерархия

1. Баланс и основная кнопка `Сканировать чек`.
2. Награда-цель:
   - название;
   - прогресс;
   - сколько баллов осталось.
3. Лотерейные билеты:
   - количество;
   - ближайший розыгрыш;
   - приз.
4. Возможность дня:
   - выгода;
   - срок;
   - партнер;
   - условие.
5. Краткая последняя активность.

## Primary CTA

`Сканировать чек` — всегда доступна без прокрутки.

## Состояния

- новый пользователь без чеков;
- нет выбранной награды;
- активная цель;
- цель достигнута;
- нет активного розыгрыша;
- нет возможности дня;
- чек обрабатывается;
- есть отклоненный чек;
- offline.

## Поведение

- после подтверждения чека карточки прогресса обновляются с понятной анимацией;
- достигнутая цель становится главным блоком до получения награды;
- возможность дня не должна визуально перекрывать основной scan CTA.

## Аналитика

- `home_viewed`;
- `scan_tapped_from_home`;
- `goal_card_opened`;
- `lottery_card_opened`;
- `daily_opportunity_opened`;
- `goal_completion_viewed`.

## Ограничения

- никакой бесконечной рекламной ленты;
- не более трех основных целей;
- пользователь должен понять свой текущий прогресс за 3 секунды.
