# Screen Specification — Receipt Result

## Цель

Показать немедленную и понятную ценность подтвержденного чека.

## Основные блоки

- статус чека;
- продавец, дата и сумма;
- начисленные баллы;
- выданные билеты;
- изменение прогресса к цели;
- примененная партнерская акция;
- CTA оставить отзыв;
- CTA сканировать следующий чек.

## Успешный результат

Пользователь должен увидеть не только `чек принят`, а конкретное изменение:

> +45 баллов. До выбранной награды осталось 18.

## Статусы

- processing;
- verified;
- verified with corrected data;
- pending manual review;
- needs rescan;
- duplicate;
- rejected;
- suspicious;
- partner bonus applied;
- reward goal completed.

## Ошибки

Для каждого отказа показывается:

1. понятная причина;
2. что пользователь может сделать;
3. возможность отправить на пересмотр, если допустимо.

## Аналитика

- `receipt_result_viewed`;
- `receipt_verified`;
- `receipt_rejected`;
- `manual_review_requested`;
- `review_cta_tapped`;
- `scan_another_tapped`;
- `goal_completed_from_receipt`.

## UX-правила

- сначала выгода, затем технические данные;
- нельзя скрывать причину отказа;
- при пакетном сканировании доступен переход к следующему результату и итог сессии.
