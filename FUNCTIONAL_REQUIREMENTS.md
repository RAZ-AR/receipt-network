# Functional Requirements

## FR-1. Регистрация и доступ

- Система должна поддерживать регистрацию по телефону или внешнему провайдеру.
- Система должна создавать уникальный User ID.
- Система должна сохранять согласия.
- Пользователь должен иметь возможность удалить аккаунт.

## FR-2. Загрузка чека

- Пользователь должен иметь возможность сфотографировать чек.
- Пользователь должен иметь возможность выбрать изображение из галереи.
- Система должна проверить качество изображения до отправки.
- Пользователь должен видеть статус обработки.

## FR-3. Распознавание

Система должна извлекать:

- продавца;
- адрес;
- дату;
- время;
- сумму;
- валюту;
- номер чека;
- фискальные данные;
- позиции, если возможно.

## FR-4. Проверка

Система должна:

- проверять обязательные поля;
- искать дубликаты;
- учитывать срок загрузки;
- рассчитывать fraud score;
- отправлять сомнительный чек на ручную проверку;
- сохранять причину отказа.

## FR-5. Баллы

- После `ReceiptVerified` начисляются баллы.
- Операция должна быть идемпотентной.
- Пользователь должен видеть историю начислений.
- Отмена чека должна отменять начисление.
- Баланс не может быть отрицательным.

## FR-6. Билеты

- На каждый допустимый подтвержденный чек создается билет.
- Билет связан с пользователем, чеком и розыгрышем.
- Пользователь видит свои активные билеты.
- Недействительный чек аннулирует билет.

## FR-7. Награды

- Пользователь видит доступные награды.
- Система проверяет баланс и наличие.
- Баллы резервируются при оформлении.
- Награда имеет срок действия.
- Использование подтверждается кодом или QR.

## FR-8. Отзывы

- Отзыв связан с подтвержденным чеком.
- Бизнес может ответить.
- Пользователь может пожаловаться.
- Награда за отзыв не зависит от оценки.

## FR-9. Бизнес-кабинет

- Бизнес может управлять карточкой.
- Бизнес может добавлять филиалы.
- Бизнес может создавать награды.
- Бизнес видит агрегированную аналитику.
- Бизнес может отвечать на отзывы.

## FR-10. Админка

- Администратор может управлять пользователями.
- Администратор может проверять чеки.
- Администратор может управлять розыгрышами.
- Администратор может корректировать баллы с обязательным аудитом.
- Администратор может блокировать мошеннические аккаунты.

## FR-11. Цели пользователя

- Пользователь может выбрать одну награду как активную цель.
- Home показывает, сколько баллов осталось.
- После начисления прогресс обновляется.
- На Home не более трех основных целей.

## FR-12. Возможность дня

- Поддерживается одна приоритетная возможность дня.
- Она имеет срок, условие, выгоду и лимиты.
- Фиксируются просмотр, активация и подтвержденная покупка.

## FR-13. Пакетное сканирование

- Можно последовательно загрузить несколько чеков.
- Показывается очередь обработки.
- Результат доступен по каждому чеку и суммарно.

## FR-14. Атрибуция партнерской кампании

- Кампания связывается с подтвержденным чеком.
- Система различает нового и повторного клиента.
- Бизнес видит подтвержденные покупки, а не только показы и клики.

## FR-15. Целевые уведомления

- Поддерживается прогресс к награде.
- Поддерживается срок розыгрыша.
- Поддерживается возможность дня.
- Действуют частотные лимиты.
