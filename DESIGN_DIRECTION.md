# Design Direction

**Статус:** принято
**Стиль:** Soft Glass (неоморфизм + жидкое стекло + голографические градиенты)
**Референс:** Figma Community «Soft GUI / Neumorphic Unicorn» (martina.wolna) + liquid-glass dock

Визуальный язык пользовательского приложения Beleg. Продуктовые копирайты — на сербской латинице. Документ фиксирует токены и правила, чтобы разработка началась по спеке, а не восстанавливала стиль по макетам.

---

## 1. Принципы

1. Светлая неоморфная подложка — весь UI «выдавлен» из одной поверхности двойной тенью.
2. Голографическая аура используется **только** за герой-элементами (кольцо цели, круг результата, аватар, орб онбординга). Не украшает фон.
3. Цвет несёт смысл: пастельные градиенты — ценность и прогресс; семантические цвета — состояния (принято / решаемо / отклонено).
4. Крупные табличные числа — главный элемент экрана (баланс, поени, прогресс).
5. Нижний док — жидкое стекло: полупрозрачный blur со стеклянными кнопками, центральный скан в радужном кольце.
6. Тёмная тема — **отдельная задача**. Неоморфизм живёт на светлой подложке; MVP-экраны намеренно light-only.

## 2. Цветовые токены

### Поверхность (неоморфизм)
```
--bg:     #EAEDF5   /* основная подложка */
--hi:     #FFFFFF   /* блик тени: сверху-слева */
--lo:     #C3CBDE   /* тень: снизу-справа */
--track:  #DDE2EE   /* вдавленные дорожки, слайдеры */
--ink:    #3E4557   /* основной текст */
--ink-2:  #8A93A8   /* вторичный текст, лейблы */
```

### Тени (двойная — сердце стиля)
```
Выпуклая:  box-shadow: 7px 7px 16px var(--lo), -7px -7px 16px var(--hi);
Вдавленная: box-shadow: inset 4px 4px 9px rgba(195,203,222,.6), inset -4px -4px 9px #fff;
Крупная (герой): 10–12px смещение вместо 7px.
```

### Голографическая аура (unicorn)
```
conic-gradient(from 30deg, #FFC9E2, #FFE4C4, #D6F7DB, #C5E9FF, #DFCFFF, #FFC9E2)
filter: blur(22–24px); opacity: .82;
```

### Градиент кольца прогресса
```
linear #FF9ECB → #B99CFF → #7FD4F5   (розовый → лиловый → голубой)
```

### Пастельные заливки (награды, чипы, столбики)
```
pink   #FFD3E8 → #D9CBFF
blue   #C3ECFF → #A8E8B4
peach  #FFE4C4 → #FFB8A8
lav    #DFCFFF → #C3ECFF
```

### Семантика состояний (после скана)
| Состояние | Значение | Аура | Иконка-заливка | Чип bg / текст |
|---|---|---|---|---|
| OK (зелёное) | принят, поени начислены | `D6F7DB→C5E9FF→DFCFFF` | `#A8E8B4→#7FD4C0` | `#D6F0E4` / `#2E7D5B` |
| Warn (жёлтое) | решаемо (QR, срок) | `FFE4C4→FFD9B0` | `#FFD9A8→#F5BE7E` | `#FBEBCF` / `#9A6E1E` |
| Err (розовое) | отклонён (невалиден, дубликат) | `FFD3D9→FFCBE0` | `#F5B8C8→#E89AB0` | `#FADBE2` / `#A24E6C` |

## 3. Типографика

- **Шрифт:** Manrope (вариативный), начертания **500 / 700 / 800**.
- Полная поддержка сербской латиницы (č, ć, š, đ, ž) — подключать latin-ext subset.
- Числа: `font-variant-numeric: tabular-nums` везде, где идёт счёт.
- Герой-число: 44–52px / 800 / `letter-spacing: -.02em`.
- Uppercase-лейблы: 10–11px / 700 / `letter-spacing: .09em`.

## 4. Компоненты

- **Карточка (`.card`):** radius 20px, `--bg`, выпуклая тень.
- **Вдавленный блок (`.inset`):** radius 16px, вдавленная тень — для дорожек, слайдеров, стат-боксов.
- **Стеклянная CTA:** `rgba(255,255,255,.55)` + `backdrop-filter: blur(12px)` + пастельный `:before`-отлив + внутренний блик.
- **Liquid-glass dock:** `rgba(255,255,255,.42)` + `blur(18px) saturate(1.5)`, border `rgba(255,255,255,.75)`, radius 999px. Центральная кнопка скана 54px в радужном конусе с матовой стеклянной сердцевиной.
- **Билет срёчки:** карточка с двумя вырезами-перфорациями по бокам (`:before`/`:after` круги цвета фона), номер вида `BG2200777`.
- **«Гусеница» срёчек:** ряд радужных точек 11px с `margin-right:-3px`.

## 5. Карта экранов (готовый дизайн)

Onboarding → Auth → Home → Scanner → Result → [5 состояний] → Wallet → Rewards → Reward details → Lottery → History → Profile.

**Home:** хедер (имя + город + дата) → кольцо цели с аурой → 2 дашборд-карточки (срёчки, предложение дня) → интересная информация (недельный график + живой факт) → стеклянный док.

**Состояния результата:** успех (зелёный, поени + номер срёчки) + 4 отказа — QR не распознан (жёлтое), невалиден (розовое), старше 3 дней (жёлтое), дубликат (розовое). Каждое состояние даёт выход — повторный скан, ручной ввод или поддержку.

## 6. Открытые задачи дизайна

- Экран отзыва (Utisak после чека).
- Экран уведомлений.
- Редемпшн награды (QR/код на кассе).
- Пакетное сканирование (очередь).
- Бизнес-портал (отдельный визуальный трек).
- Тёмная тема (если понадобится — отдельная адаптация неоморфизма).
